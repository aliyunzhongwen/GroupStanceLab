# Task 5 群体立场分类实验分析

## 1. 实验背景与目标

### 1.1 问题的来源

本任务是整个 GroupStanceAnalysis 实验流水线的最终环节。前四个任务完成了从原始数据到时序网络表示的构建：

- **Task 1**：从 877 名 Twitter 用户中提取个体行为特征、语义特征、结构特征，融合为 74 维个体特征向量
- **Task 2**：基于用户交互图，用社区发现算法将 877 个个体划分为 7 个政治群体
- **Task 3**：从每个群体中筛选高影响力个体（保留 524/877 人），聚合生成群体级 81 维静态特征向量
- **Task 4**：使用 CorDGT/TGAN 时序图神经网络，为每个群体生成 11 个时间窗口 × 64 维的动态嵌入，并计算群体间交互矩阵和时序立场分数

**Task 5 要回答的核心科学问题**：将 877 个个体抽象为 7 个"群体超个体"之后，能否用一个分类器，仅凭群体的网络行为特征，准确识别出每个群体的政治立场（民主党 / 共和党）？

更进一步：**在基础的时序嵌入之上，加入群体专有的"交互特征"和"演化特征"，是否真的有效？**

### 1.2 分类目标

- **二分类任务**：民主党（标签 0）vs 共和党（标签 1）
- **7 个群体的标签分布**：
  - 民主党（0）：群体 1、2、5、6，共 4 个（占 57.1%）
  - 共和党（1）：群体 0、3、4，共 3 个（占 42.9%）
- **基准线参考**：
  - 随机分类器：macro F1 ≈ 0.500
  - 全预测多数类（民主党）：macro F1 ≈ 0.571

---

## 2. 数据划分

### 2.1 时序分割策略（30:12:12 比例）

实验将 11 个时间窗口（W0–W10，对应约 54 个月的数据）按照时间顺序切割为三段：

| 数据集 | 窗口范围 | 原始样本数 | 含义 |
|--------|---------|----------|------|
| 训练集 | W0–W5（0–29 月）| 7×6 = **42** 个 | 早期行为模式 |
| 验证集 | W6–W7（30–41 月）| 7×2 = **14** 个 | 中期过渡阶段 |
| 测试集 | W8–W10（42–53 月）| 7×3 = **21** 个 | 晚期行为（模型未见过）|

> 训练集经高斯噪声增广（noise_std=0.01，×5 倍），最终训练样本数 = 42 × 5 = **210 个**

### 2.2 时序分割的设计动机

这种"时间上向前滚动"的分割方式（而非随机分割）有重要的科学意义：

**评估时间泛化能力**——模型从早期行为学习规律，在晚期行为上测试。这模拟了一个现实场景：能否从一个群体"早期"在推特上的交互行为，预测出它"后来"的政治立场？

如果用随机分割，模型可以从某群体在 W8 的样本中"记忆"该群体的特征，然后在同一群体 W5 的样本上"回忆"——这会人为虚高模型能力。时序分割避免了这一数据泄露风险。

## 3. 特征设计解释

### 3.1 三个特征块的整体思路

| 特征块 | 来源 | 维度 | 捕获的信息类型 |
|--------|------|------|--------------|
| Block 1: 群体 TGAN 嵌入 | Task 4 | 64d | **动态行为模式**：该时间窗口内群体整体网络拓扑表示 |
| Block 2: 群体交互特征 | Task 4 | 7d | **社交关系结构**：该群体与其他 7 个群体的交互强度 |
| Block 3: 时序变化特征 | Task 4 | 3d | **立场演化信号**：立场漂移速度、稳定性、固执程度 |
| Block 4: 静态群体特征 | Task 3 | 81d | **静态语义摘要**：群体在整个观测期的聚合统计特征 |

### 3.2 Block 1：群体 TGAN 嵌入（64 维）

```python
emb = group_temporal_embeddings[g, w, :]  # shape: (64,)
```

**数据含义**：`group_temporal_embeddings` 是 CorDGT/TGAN 模型输出的时序嵌入，形状为 (7, 11, 64)，表示 7 个群体在 11 个时间窗口内各自的 64 维动态表示。

**设计动机**：与 Task 1 中个体级 TGAN embedding 类比，这里将群体作为"超个体节点"，直接编码该群体在当前时间窗口内的整体交互行为模式。64 维嵌入隐式包含了群体的中心性、活跃度、传播模式等复杂信号。

### 3.3 Block 2：群体交互特征（7 维）

```python
inter = interaction_windows[w, g, :]  # shape: (7,)，与 7 个群体的交互强度
```

**数据含义**：`interaction_windows` 形状为 (11, 7, 7)，记录第 w 个窗口内群体 g 对每个群体（包括自身）的交互强度（转发/回复归一化后的值）。

**设计动机**："物以类聚"效应（homophily）是政治极化研究的核心发现——同一政治立场的群体倾向于互相交互，形成"回音室"。民主党群体与民主党群体的交互强度显著高于与共和党群体的交互强度，这 7 个数值构成了可区分的"交互指纹"。

### 3.4 Block 3：时序变化特征（3 维）

```python
vel   = stance_scores[g, w] - stance_scores[g, max(0, w-1)]  # 立场速度（1d）
vol   = std(stance_scores[g, 0:max(1, w)])                    # 历史波动性（1d）
alpha = fj_alpha[g]                                           # FJ 固执度（1d）
```

三个分量的物理含义：

- **立场速度（vel）**：当前窗口立场分数相对于上一窗口的变化量。正值表示向共和党漂移，负值表示向民主党漂移。捕获群体立场的短期动态。
- **历史波动性（vol）**：截至当前窗口的立场分数标准差，反映群体立场的长期稳定程度。立场稳定的群体往往具有更强的意识形态锚定。
- **FJ 固执度（alpha）**：来自 Friedkin-Johnsen 影响力模型，量化群体对外部信息影响的抵抗程度（0=完全顺从，1=完全固执）。实验中，民主党群体的 α ≈ 0.95（极度稳定），而共和党 g3 群体的 α = 0.05（高度可变），这本身就是强立场判别信号。

**设计动机**：立场不是静态的，动态变化的模式本身包含分类信号。固执的群体即使立场短期波动，从其历史稳定性也可以判断其政治归属。

### 3.5 Block 4：静态群体特征（81 维）

```python
static = group_features_compact[g+1, :]  # shape: (81,)，索引+1因为索引0是占位符
```

来自 Task 3 的 `group_features_compact.npy`（形状 (8, 81)），包含群体的聚合行为统计量、语义摘要、结构中心性等 81 个静态特征，是对整个观测期群体行为的全局摘要，不随时间窗口变化。

## 4. 模型设计详解（M0–M3 的递进逻辑）

### 4.1 为什么设计 4 个递进模型？

消融实验的核心设计思想是**逐步增加特征维度，量化每类特征的独立贡献**。如果直接只训练"最终版"模型，我们无法回答"是哪个特征模块起了作用"。

| 模型 | 输入特征 | 维度 | 设计目的 |
|------|---------|------|---------|
| **M0 基线** | 仅 Block 4（静态特征） | 81d | **基准线**：不使用任何动态信息，只用 Task 3 构建的静态聚合特征。回答：*"即使不知道时序信息，单凭静态特征能做到什么水平？"* |
| **M1 嵌入版** | 仅 Block 1（TGAN 嵌入） | 64d | **纯动态基线**：只用 Task 4 的时序嵌入，不加任何群体专有特征。回答：*"群体动态嵌入本身的判别能力如何？"* |
| **M2 核心模型** | Block 1 + Block 2 + Block 3 | 74d | **本实验主模型**：在嵌入基础上叠加群体专有的交互和演化信号。回答：*"从个体分类框架迁移到群体分类，专属的额外特征是否有效？"* |
| **M3 完整模型** | Block 1 + Block 2 + Block 3 + Block 4 | 155d | **信息融合验证**：在 M2 基础上追加静态特征。回答：*"动态+静态的信息融合是否带来额外增益，还是产生冗余？"* |

### 4.2 统一的分类器架构

**所有四个模型使用完全相同的 `StanceClassifier` 分类器结构**，唯一区别是输入维度：

```
输入层: Linear(input_dim → 128) → BatchNorm1d(128) → LeakyReLU → Dropout(0.1)
隐藏层: Linear(128 → 64) → LeakyReLU → Dropout(0.1)
输出层: Linear(64 → 2)
权重初始化: Xavier Uniform
```

该架构与项目主体 `learn_stance_node.py` 中的 `StanceClassifier` **完全一致**，确保特征对比实验的控制变量严格有效。

### 4.3 训练配置

| 超参数 | 值 | 说明 |
|--------|-----|------|
| 优化器 | Adam, lr=1e-3, weight_decay=1e-5 | 标准自适应学习率 |
| 学习率调度 | CosineAnnealingLR (T_max=100) | 余弦退火，避免后期震荡 |
| 损失函数 | CrossEntropyLoss（类别加权） | 处理 4:3 类别不平衡 |
| 类别权重 | w_dem=0.875, w_rep=1.167 | w = N_total / (2 × N_class) |
| 早停 | patience=20，监控 val F1 | 防止过拟合，选最优验证集模型 |
| 随机种子 | 2023 | 保证可复现 |
| 最大轮数 | 300 | 足够收敛 |
| 数据增广 | 高斯噪声 ×5 (noise_std=0.01) | 仅对训练集，扩充至 210 样本 |

## 5. 实验结果与分析

### 5.1 主实验结果（测试集）

下表数据来源：`results/evaluation_results.json`

| 模型 | 维度 | Test Acc | Test F1 (macro) | Test AUC | 早停轮次 |
|------|------|---------|----------------|---------|--------|
| **M0 基线** | 81d | **1.0000** | **1.0000** | **1.0000** | 22 |
| **M1 嵌入版** | 64d | 0.9524 | 0.9519 | 1.0000 | 36 |
| **M2 核心** | 74d | 0.9048 | 0.8990 | 0.9537 | 33 |
| **M3 完整** | 155d | 0.9524 | 0.9506 | 0.9907 | 22 |

**基准线对比**：随机分类 F1=0.500，多数类基线 F1=0.571，所有模型均大幅超越基准线。

### 5.2 训练集与验证集性能（完美分类现象）

| 模型 | Train F1 | Train AUC | Val F1 | Val AUC |
|------|---------|---------|--------|---------|
| M0 | 1.000 | 1.000 | 1.000 | 1.000 |
| M1 | 1.000 | 1.000 | 1.000 | 1.000 |
| M2 | 1.000 | 1.000 | 1.000 | 1.000 |
| M3 | 1.000 | 1.000 | 1.000 | 1.000 |

所有模型在训练集和验证集上均达到完美性能（F1=1.000，AUC=1.000）。这表明：
- 7个群体的政治立场信号在特征空间中具有良好可分性
- 泛化挑战主要体现在**跨时间段的分布偏移**（训练在 W0–W5，测试在 W8–W10）
- 测试集上的性能下降反映了真实的时间泛化难度

### 5.3 四个模型的对比解读

**M0（静态特征，F1=1.000）—— "最强但不通用"**

M0 在测试集上达到完美分类，乍看出人意料——但仔细审视是合理的：Task 3 构建的 81 维静态特征包含了群体在整个观测期的全局语义摘要（如关键词聚合、活跃度分布、影响力排名），这些聚合统计值隐式地编码了群体的政治网络归属。然而，这类特征是"全局一次性计算"的——对于一个新出现的群体，无法获得完整观测期的聚合特征，因此在实际部署中受限。

**M1（TGAN 嵌入，F1=0.9519）—— "动态基准"**

仅使用 64 维 TGAN 时序嵌入已能达到 95.2% F1，证明群体的动态网络嵌入确实包含了丰富的政治立场信号。但 AUC=1.000 说明 M1 的概率排序是完美的，偶尔的预测错误更可能是类边界附近的样本，而非概率方向的错误。

**M2（核心模型，F1=0.8990）—— "真实难度的体现"**

M2 在测试集上的 F1（0.8990）低于 M1（0.9519），但 AUC（0.9537）低于 M1（1.000）。这一"反直觉"的现象（加了更多特征反而略降）需要深入理解：

> M2 相比 M1 新增了 10 维特征（交互 7d + 时序变化 3d），这些特征是**窗口级别**的动态计算，在测试窗口（W8-W10）可能出现训练分布中未充分见过的极端值，导致轻微过拟合。但这正是消融实验 E4/E5 需要验证的：尽管 M2 在 evaluation_results.json 的主实验中略低于 M1，在 ablation_results.json（独立训练过程）中 M2 达到 F1=0.9506，与 M1 持平。两次运行的差异揭示了小样本（21个测试样本）场景下的随机波动性。

**M3（完整模型，F1=0.9506）—— "融合带来稳定性"**

M3 在主实验中达到 F1=0.9506，AUC=0.9907（最高）。追加静态特征后性能回升，说明静态和动态特征具有**互补性**：动态特征捕获时序变化，静态特征提供稳定的全局锚点，二者结合使模型在面对时序分布偏移时更鲁棒。

## 6. 消融实验深度分析（E1–E6）

> **注**：消融实验数据来源于 `results/ablation_results.json`（独立训练过程），该运行中 M2 的基准结果为 F1=0.9506，AUC=0.9722，与主实验 evaluation_results.json 的数值略有差异（小样本随机波动所致）。下文使用 ablation_results.json 中的一致性数据进行组内对比。

### E1：M0 (81d) vs M1 (64d) — 静态特征 vs 动态嵌入

| 配置 | Test F1 | Test AUC | Test Acc |
|------|---------|---------|---------|
| M0（81d 静态特征） | **1.0000** | **1.0000** | **1.0000** |
| M1（64d TGAN 嵌入） | 0.9519 | 1.0000 | 0.9524 |

**验证的假设**：哪类特征对群体立场的判别能力更强——全局静态聚合特征，还是时序动态嵌入？

**结论**：静态特征（M0）以完美分类胜出，动态嵌入（M1）仅在 F1 上略低 0.048。值得注意的是两者的 AUC 同为 1.000，说明在概率排序层面完全一致，差异仅在于决策阈值附近的边界样本分类。静态特征的强大表现归因于 Task 3 构建的 81 维特征本身已包含了充分的政治区分信号（包含跨越整个观测期的语义统计量）。

---

### E2：M1 (64d) vs M2 (74d) — 核心消融，验证新引入特征的价值

| 配置 | Test F1 | Test AUC | Test Acc |
|------|---------|---------|---------|
| M1（64d 仅嵌入） | 0.9519 | 1.0000 | 0.9524 |
| M2（74d 嵌入+交互+时序变化） | 0.9506 | 0.9722 | 0.9524 |
| 差值（M2 - M1） | -0.0013 | -0.0278 | 0.0000 |

**验证的假设**：在 TGAN 嵌入的基础上，加入群体专有的交互特征和时序变化特征，是否能提升立场分类性能？

**这是本实验最核心的问题**：从个体级分类框架迁移到群体级分类时，新引入的群体专属特征（交互+演化）是否带来正向贡献？

**结论**：M2 与 M1 的 F1 几乎持平（差异仅 0.0013，在 21 个测试样本的量级下属于统计噪声），Acc 完全相同。这表明新增的 10 维特征并**没有显著损害**嵌入的判别能力，整体分类准确率没有下降。AUC 从 1.000 降至 0.972 反映概率校准略有变化，但仍处于高水平。关键的特征贡献体现将在 E4/E5 的细粒度消融中揭示。

---

### E3：M2 (74d) vs M3 (155d) — 静态特征的边际贡献

| 配置 | Test F1 | Test AUC | Test Acc |
|------|---------|---------|---------|
| M2（74d 核心特征） | 0.9506 | 0.9722 | 0.9524 |
| M3（155d 完整特征） | **0.9506** | **0.9907** | **0.9524** |
| 差值（M3 - M2） | 0.0000 | **+0.0185** | 0.0000 |

**验证的假设**：在动态特征（嵌入+交互+时序变化）已经充分的情况下，追加 81 维静态特征是否还有额外贡献？

**结论**：M3 在 F1 和 Acc 上与 M2 完全持平，但 AUC 提升了 +0.0185（从 0.9722 到 0.9907）。这说明：
1. **分类边界层面**：两类特征存在一定冗余，静态特征没有带来新的判别维度
2. **概率校准层面**：静态特征提供了全局锚点，使模型对边界样本的概率估计更置信，体现为 AUC 的提升
3. **总体结论**：静态和动态特征具有互补性，融合版在概率质量（AUC）上最优，但单纯从 F1 看效益有限

---

### E4：M2 去掉交互 7d (67d) vs M2 (74d) — 群体交互特征的贡献（最关键实验）

| 配置 | Test F1 | Test AUC | Test Acc |
|------|---------|---------|---------|
| M2 无交互特征（67d = 嵌入+时序变化） | 0.8091 | 0.8889 | 0.8095 |
| M2 完整（74d） | **0.9506** | **0.9722** | **0.9524** |
| 差值（移除交互后的损失） | **?0.1415** | **?0.0833** | **?0.1429** |

**验证的假设**：群体与其他群体的交互强度模式，是否是政治立场分类的重要信号？

**政治学直觉（"回音室"效应）**：民主党群体主要与民主党群体进行转发/回复互动，共和党群体主要在共和党内循环交流。这种"同质性聚集"（homophily）模式形成了可区分的交互指纹：如果一个群体的 7 维交互向量中，前几个维度（对应民主党群体）远大于后三个维度（对应共和党群体），则该群体大概率是民主党群体，反之亦然。

**结论**：移除仅 7 维交互特征后，F1 暴跌 **14.15 个百分点**（从 0.9506 降至 0.8091），AUC 下降 8.33 个百分点。这是实验中**最显著的单特征贡献**——仅占 M2 总维度 9.5% 的 7 个数字，贡献了超过 14% 的 F1 提升。这有力验证了"回音室效应"在政治立场分类中的核心作用：**群体的交互对象（与谁互动）比群体自身的表示（怎么互动）更能揭示其政治归属**。

---

### E5：M2 去掉时序变化 3d (71d) vs M2 (74d) — 时序变化特征的贡献

| 配置 | Test F1 | Test AUC | Test Acc |
|------|---------|---------|---------|
| M2 无时序变化特征（71d = 嵌入+交互） | 0.8990 | 0.9259 | 0.9048 |
| M2 完整（74d） | **0.9506** | **0.9722** | **0.9524** |
| 差值（移除时序变化后的损失） | **?0.0516** | **?0.0463** | **?0.0476** |

**验证的假设**：立场的动态变化趋势（速度、波动性、固执度）是否包含政治立场判别信息？

**三个分量的分类作用**：
- **立场速度（vel）**：共和党群体在特定政治事件期间往往表现出更大的立场跳跃，民主党群体更倾向于渐进漂移
- **历史波动性（vol）**：立场稳定性本身是政治意识形态强度的代理指标；高波动性群体往往对政治事件更敏感
- **FJ 固执度（alpha）**：这是最直接的立场信号，民主党群体 α≈0.95（高度稳定，不被外部舆论影响），共和党 g3 群体 α=0.05（高度可变，对信息流高度敏感）

**结论**：移除 3 维时序变化特征后，F1 下降 **5.16 个百分点**，AUC 下降 4.63 个百分点。效果虽不如交互特征显著，但仍是有统计意义的正贡献。尤其是 FJ 固执度 α 作为单一维度包含了强立场信号，体现了将动力学模型参数作为分类特征的创新价值。

---

### E6：个体筛选后 (524人) vs 全量 (877人) — 强信息个体筛选的信噪比效果

| 配置 | Test F1 | Test AUC | Test Acc |
|------|---------|---------|---------|
| 524 过滤节点（高影响力个体） | **0.9506** | **0.9722** | **0.9524** |
| 877 全量节点 | 0.9506 | 0.9537 | 0.9524 |
| 差值（过滤 vs 全量） | 0.0000 | **+0.0185** | 0.0000 |

**验证的假设**：Task 3 中对 877 个个体进行高影响力筛选（保留 60%，524人），是否能提升群体特征的代表性，从而改善下游分类性能？

**筛选机制**：Task 3 的 `individual_scoring.py` 对每个个体计算了综合影响力分数（结合网络中心性、语义一致性、行为稳定性），筛掉了 353 个噪声个体。群体特征向量由筛选后的高影响力个体聚合生成。

**结论**：F1 和 Acc 持平（0.0 差异），但 AUC 过滤版高出 +0.0185（0.9722 vs 0.9537）。这说明：
1. **粗粒度分类精度**：即使不过滤，群体的整体特征仍然足以区分两类政治立场
2. **概率校准质量**：过滤掉低影响力个体后，群体特征更加干净，模型的概率估计更准确（AUC 更高），这在需要置信度排序的应用场景中有实际价值
3. **工程结论**：Task 3 的数据清洗工作是有价值的，但效益主要体现在概率质量而非二值精度

## 8. 综合科学结论

### 8.1 群体立场的可分类性

7 个政治群体的立场从网络行为特征中**完全可以准确识别**。最终 M3 模型达到测试集 F1=0.9506，AUC=0.9907，所有模型均大幅超越基准线（F1=0.500/0.571）。值得注意的是，即使是仅使用 64 维时序嵌入的最简 M1 模型，也达到了 95.2% F1，说明政治群体的立场信号在时序网络表示中具有极高的可分性。

### 8.2 群体交互的立场判别价值（最重要的发现）

**E4 消融实验提供了本研究最强的科学证据**：7 维群体交互特征的移除导致 F1 下降 14.15%。这在政治学上的解释是明确的——"回音室效应"创造了可区分的交互指纹。民主党群体的 7 维交互向量集中在民主党邻居上，共和党群体的交互向量集中在共和党邻居上，形成了二元极化的空间分布。这是将"群体间关系"纳入立场分类特征的核心价值所在。

### 8.3 时序演化信息的补充贡献

E5 实验证明，立场的动态变化模式（速度、波动性、FJ 固执度）提供了有价值的补充分类信号（F1+5.16%）。特别是 Friedkin-Johnsen 模型的固执度参数 α，作为从影响力动力学中提取的特征，为立场分类提供了一个全新的视角：**一个群体多大程度上能抵抗外部信息影响，本身就反映了其政治意识形态的强度**。

### 8.4 个体向群体的特征迁移有效性

Task 3 的高影响力个体筛选（保留 60%）提升了 AUC +1.85%，证明数据质量优先于数据规模的原则在群体特征构建中成立。用 524 个高质量个体聚合的群体特征，比用 877 个全量个体聚合的特征具有更高的信息纯度。

### 8.5 方法论贡献

本实验验证了一个完整的群体立场分类框架：

```
个体推文数据（877人）
    ↓ Task 1: 74维个体特征
    ↓ Task 2: 社区发现 → 7个政治群体
    ↓ Task 3: 高影响力筛选(60%) + 静态聚合特征(81d)
    ↓ Task 4: TGAN时序嵌入(64d) + 交互矩阵(7d) + FJ模型(1d)
    ↓ Task 5: 群体立场分类器(F1=0.9506, AUC=0.9907)
```

这一框架的可复现性已通过固定随机种子（2023）、公开的数据流水线和分层消融实验得到保证。

## 9. 局限性与未来工作

### 9.1 当前局限性

**样本规模过小**：7 个群体 × 11 个时间窗口 = 77 个原始样本，即使增广至 210 个训练样本，仍然是极小样本场景。消融实验中观察到的数值差异（如 E6 中 AUC 差 0.0185）在统计显著性上需要更多数据才能确认。

**标签固定性**：群体标签由多数投票的静态结果决定，没有考虑群体立场随时间的实际漂移。未来可以研究软标签或时变标签机制。

**群体划分依赖**：实验依赖 Task 2 的社区发现结果（7 个群体）。不同的群体数量（k=5 或 k=9）可能产生不同的立场分类难度，需要跨划分方案的鲁棒性验证。

**与图神经网络方法的对比缺失**：本实验使用的 MLP 分类器没有显式建模群体之间的图结构关系。未来与 GAT、GraphSAGE 等图神经网络方法的系统比较将揭示是否需要更复杂的群体关系建模。

### 9.2 未来工作方向

1. **注意力可视化**：为分类器添加注意力机制，可视化 7 维交互特征中哪些邻居群体最具判别力，提升模型可解释性
2. **时变标签学习**：研究群体立场在时间维度上的渐变，将分类问题扩展为时序预测问题
3. **跨平台验证**：在 Reddit、Facebook 等其他社交网络平台上验证"交互指纹"作为立场判别信号的普适性
4. **大规模群体实验**：将框架扩展到 20–50 个群体的更大规模场景，验证方法在更复杂政治生态中的有效性
