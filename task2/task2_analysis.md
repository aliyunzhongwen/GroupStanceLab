# Task 2 群体划分实验分析

## 1. 实验目标

对 877 个 Twitter 政治用户（民主党 471 人 + 共和党 406 人）进行群体划分，目标是同时满足以下三方面要求：

1. **结构合理性**：划分结果在交互网络上具有较高的模块度（Modularity），群体内部连接稠密、群体之间连接稀疏。
2. **立场一致性**：每个群体内部成员的政治立场应尽可能一致（高纯度 Purity 与较高的 NMI）。
3. **群体平衡性**：避免出现"超大群体 + 多个孤立小群体"的退化解，要求群体规模分布相对均衡且每个群体至少 20 人，群体数控制在 8–15 之间（最终最优解为 7 个群体）。

整体思路：以 Task 1 输出的特征 + 原始交互边表为输入，构建静态加权交互图，再对比 Louvain、谱聚类、K-Means、特征+图融合等多种社区发现算法，并引入"小群体合并 + 大群体拆分"的平衡后处理，依据综合评分挑选最优方案。

## 3. 实验方法

### 3.1 静态图构建（`build_static_graph.py`）

**核心步骤**：

1. 从 `ml_twitter.csv` 读取原始边记录 `(u, i, ts, label, idx)`，每行表示一次交互；
2. 调用 `build_weighted_graph(edges_df, directed)` 按 `(u, i)` 聚合：

   ```python
   edge_counts = edges_df.groupby(['u', 'i']).size().reset_index(name='weight')
   ```

   即同一对节点之间的多次交互被合并为一条边，**权重 = 累计交互次数**。
3. 节点空间固定为 `range(1, 878)`，确保即便孤立节点也保留在图中，避免后续指标计算时索引错位。
4. 分别构建：综合有向图、综合无向图、转发图、回复图，并以 `pickle` 序列化。

**边权重策略**：累计计数（cumulative count），简单、可解释，且保留了交互强度信息。在无向版本中，`(u→i)` 与 `(i→u)` 的权重相加。

### 3.2 社区发现算法对比（`community_detection_balanced.py`）

共评估 5 大类、21 种参数组合的方法。所有方法均在 877 节点的综合无向加权图（22,099 条边）上运行。

#### 3.2.1 Louvain 多分辨率 + 小群体合并

- **原理**：基于贪心模块度优化的多层次社区发现算法，分辨率参数 `resolution` 越高、产生的群体越多。
- **参数**：`resolution ∈ {1.5, 2.0, 2.5}`；后处理 `merge_min=20`。
- **后处理**：对成员数 < 20 的小群体，依据"边权重连接强度"合并到最相关的大群体。

#### 3.2.2 层次 Louvain（HierLouvain）

- **原理**：先用低分辨率得到粗粒度群体，再对成员数 > 150 的大群体内部递归运行 Louvain（`resolution=1.5`），之后再合并 < 20 的小群体。
- **参数**：`res1 ∈ {0.8, 1.0}`，`split_threshold=150`。

#### 3.2.3 谱聚类 + 平衡后处理（Spectral + Balance）

- **原理**：将加权邻接矩阵作为相似度矩阵（`affinity='precomputed'`），通过拉普拉斯算子的低阶特征向量将节点嵌入欧氏空间，再做 K-Means。
- **参数**：`K ∈ {6, 8, 10, 12, 15}`；后处理同时执行 `merge<20` 与 `split>200`。
- **拆分策略**：对 > 200 人的大群体，根据节点特征向量做内部 K-Means 子聚类，子簇数 `n_sub = clip(len/100, 2, len/20)`。

#### 3.2.4 基于特征的 K-Means（KMeans / MiniBatchKMeans + Balance）

- **原理**：直接在 Task 1 输出的紧凑特征 `individual_features_compact.npy`（81 维）上做 K-Means 聚类，**忽略图结构**。
- **参数**：`K ∈ {6, 8, 10, 12}`，分别使用标准 KMeans 与 MiniBatchKMeans。
- **后处理**：与谱聚类一致的合并 / 拆分。

#### 3.2.5 图 + 特征融合（Hybrid）

- **原理**：构建融合相似度矩阵 `S = 0.6 × adj_norm + 0.4 × cosine_sim(features)`，再做谱聚类。
- **参数**：`alpha=0.6`（图权），`K ∈ {8, 10, 12}`。

### 3.3 评价指标体系

| 指标 | 含义 |
|---|---|
| 模块度（Modularity） | 群体内部边权占比与随机期望之差，越大越好 |
| NMI | 划分结果与立场标签的归一化互信息，越大越好 |
| 纯度（Purity） | 每个群体多数立场占比的加权平均，越大越好 |
| 平衡性分数 | `1 ? (max_size ? min_size) / N`，越大越好 |
| 基尼系数 | 群体规模分布不平等度，越小越好 |
| 有效群体比例 | `≥20` 人群体数 / 总群体数，越大越好 |

**综合评分公式**（对模块度、NMI、纯度先做 Min-Max 归一化）：

```
score = 0.25·mod_norm + 0.25·NMI_norm + 0.20·purity_norm
      + 0.15·balance + 0.15·(1 ? gini)
```

### 3.4 平衡性优化

#### 3.4.1 初始版本暴露的问题

未做平衡约束时，最佳谱聚类（`K=4`）的群体大小分布为 **`[526, 347, 2, 2]`**：

- 最大群体 526 人，几乎覆盖民主党全部用户；
- 最小群体仅 2 人，无法支撑后续"超级个体"特征构建；
- 平衡性分数 = 0.4025，基尼系数 = 0.5465，有效群体比例仅 0.5；
- NMI 形式上较高（因二分聚类几乎对齐立场），但群体粒度过粗，丧失结构信息。

#### 3.4.2 平衡性约束的两步后处理

**步骤 A：合并小群体（`merge_small_groups`，`min_size=20`）**

```
while 存在成员数 < 20 的小群体:
    取最小的小群体 c_small；
    对每个其它群体 c'，统计 c_small 与 c' 之间的边权连接强度；
    将 c_small 合并到连接强度最大的群体（若无连接，则合并到最大群体）。
```

**步骤 B：拆分大群体（`split_large_groups`，`max_size=200`）**

```
对每个成员数 > 200 的大群体 c_large:
    n_sub = clip(len(c_large) / 100, 2, len(c_large) / 20)
    在该群体的特征子矩阵上做 KMeans(n_clusters=n_sub)
    将原标签 c_large 替换为 n_sub 个新标签
```

通过"先合并、后拆分"的串联操作，所有方法的有效群体比例均提升至 **1.00**，群体数稳定在 7–15 之间。

#### 3.4.3 最终选择

综合评分排名前 5 的方案均为 Spectral + Balance 系列，最高得分为 **Spectral(K=10) + Balance**（score = 0.8854）。该方案 K 设置较高，让谱算法先做精细切分，再依靠合并步骤把小簇并入语义相近的大簇，最终落到 7 个规模适中的群体上。

## 5. 结论与讨论

### 5.1 最终方案

**Spectral(K=10) + Balance** 被选为最优方案，得到 **7 个平衡群体**：

- 综合得分 **0.8854**（21 种方法中最高）
- NMI **0.3908** 与纯度 **0.9293** 均居首位
- 模块度 0.4568，与同 K 范围的 Louvain 方法处于同一量级
- 所有 7 个群体规模 ≥ 36，平衡性分数 0.8244

### 5.2 划分合理性分析

1. **立场对齐**：7 个群体中 6 个的纯度 ≥ 86%，与"政党立场是网络主结构信号"的先验一致；
2. **群体异质保留**：群体 5（73 人，48 民 / 25 共）作为天然桥接区，避免了过度极化；
3. **群体 0 的合理性**：86.59% 共和党 + 22 名民主党，可解释为"以共和党议题为主、但与对方阵营互动密切"的政治评论或媒体子社区；
4. **无孤立小群体**：所有群体 ≥ 36 人，均能为下游 Task 3 的"超级个体"特征聚合提供足够样本。

### 5.3 方法选择的关键洞察

1. **图结构 > 特征**：在 Twitter 政治网络上，交互结构对群体边界的指示作用强于单点行为/语义特征；
2. **过度切分 + 后处理 > 直接切分**：Spectral(K=10) 切出 10 个簇再合并，效果优于 Spectral(K=7) 直接切；这是"先精细后调整"思路在社区发现中的有效体现；
3. **特征 / 图融合需谨慎**：简单线性融合（0.6G + 0.4F）在本数据上反而造成相似度退化；后续若要融合，应考虑非线性方式（如 GNN 端到端学习）。

### 5.4 局限性

1. **静态图假设**：合并所有时间窗口的交互，忽略了立场演化的时间维度；
2. **K 与阈值需要调参**：`min_size=20`、`max_size=200` 是基于经验，在不同数据集上需重新调优；
3. **纯度 ≠ 立场质量**：纯度只反映多数标签占比，未考虑边界用户的政治意义；
4. **小群体（36）的统计稳健性**：群体 4 仅 36 人，下游聚合特征的方差可能偏大。

### 5.5 改进方向

- 引入时序权重（如指数衰减）构建动态加权图；
- 结合 GNN 学习"图-特征"联合表示，再做聚类；
- 使用立场标签做半监督社区发现（如 Constraint-NMF）；
- 对桥接群体（群体 5）进行内部细分，分析其跨党派话语特征。
