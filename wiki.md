# 群体立场分析实验 Wiki

## 1. 实验概述

### 1.1 研究背景
- 个体立场检测的局限性：现有方法关注个体用户的立场分类，但忽略了群体效应
- 群体立场的意义：社交网络中用户自然形成群体（如政治派系、兴趣社区），群体立场比个体立场更稳定、更有实际意义
- 从个体到群体的挑战：如何定义群体边界、如何聚合个体特征、如何建模群体间交互

### 1.2 研究目标
- 基于个体立场数据集（877个Twitter政治用户，114,944条交互）
- 实现从个体立场检测到群体立场识别的完整流程
- 提出有效的群体划分、个体筛选、群体交互分析和群体立场分类方法

### 1.3 核心概念定义
- **个体立场**: 单个用户的政治倾向（民主党0 / 共和党1），由TGAN模型学习得到64维嵌入表示
- **群体(Group)**: 通过社区发现算法从交互网络中识别出的用户子集，具有紧密的内部交互
- **强信息个体(Key Individual)**: 在群体中具有高信息量/高影响力的用户，通过活跃度、中心性等指标筛选
- **演化分析(Evolution Analysis)**: 研究群体立场在时间维度上因群体间交互影响而产生的动态变化过程。核心问题：不同群体间存在交互，交互行为会影响群体的立场变化，这是一个随时间维度变化的动态现象
- **烟花图(Firework Diagram)**: 一种可视化工具，用来展示同一立场群体内部的信息传播扩散效果（节点为个体，从核心向外辐射）
- **演化模式(Evolution Pattern)**: 群体立场在交互影响下的变化类型，包括：趋同型(Convergence)、极化型(Polarization)、稳定型(Stability)、传染型(Contagion)
- **因果影响网络(Causal Influence Network)**: 基于Granger因果检验构建的有向图，边表示一个群体对另一个群体的立场变化具有显著因果影响
- **群体立场(Group Stance)**: 群体层面的政治倾向标签，由组成个体的立场聚合和群体交互模式共同决定
- **群体极化(Group Polarization)**: 衡量不同立场群体之间对立程度的指标。当同立场群体内交互远多于跨立场群体交互时，呈现高度极化。可用E-I Index（External-Internal Index）、交互矩阵模块度等量化
- **极小化(Minimization / Group Merging)**: 评估两个群体合并为一个群体的合理性。当群体间交互紧密、特征高度相似时，合并更合理。指标包括群体间交互密度、特征余弦相似度、合并后模块度变化。用于检验群体划分是否过度分割（over-segmentation）
- **极大化(Maximization / Group Splitting)**: 评估两个有联系但联系不紧密的群体彻底分裂为独立群体的合理性。当群体间交互稀疏、特征差异明显时，分裂更合理。指标包括群体间交互稀疏度、特征差异度、分裂后模块度变化。用于检验群体划分是否存在不充分分割（under-segmentation）

## 3. 技术路线

### 3.1 整体框架
```
个体立场数据 -> 个体特征提取 -> 群体划分 -> 个体筛选 -> 群体特征构建 -> 演化分析(Evolution Analysis) -> 群体立场分类
```

### 3.2 阶段一：个体特征提取与增强
- 利用已有TGAN模型获取个体节点嵌入（64维）
- 提取个体的结构特征（度中心性、介数中心性、聚类系数等）
- 提取个体的行为特征（发推频率、转发率、回复率、活跃时间跨度等）
- 提取个体的语义特征（BERT嵌入的统计量：均值、方差等）
- 多维度特征融合，形成完整的个体画像向量

### 3.3 阶段二：群体划分
- **基于图结构的社区发现**：
  - Louvain算法：基于模块度优化的社区检测
  - Label Propagation：基于标签传播的快速社区划分
  - Spectral Clustering：基于图拉普拉斯的谱聚类
- **基于特征的聚类**：
  - K-means / GMM：基于个体特征向量的聚类
  - 结合图结构约束的聚类方法
- **混合方法**：
  - 同时考虑图结构和特征相似度的社区发现
  - 权衡结构紧密度和特征一致性

### 3.4 阶段三：强信息个体筛选与群体特征整合
- **筛选动机**: 877个个体数量过多，存在大量低活跃度、低信息量的用户
- **筛选指标**:
  - 活跃度：推文数量、交互频率
  - 中心性：PageRank、度中心性、介数中心性
  - 信息量：推文内容丰富度、立场表达强度
  - 影响力：被转发/被回复次数
- **筛选策略**:
  - 设定阈值过滤低活跃度用户
  - 保留每个群体中的核心成员（Top-K）
  - 确保筛选后每个群体仍有足够代表性
- **群体特征整合**（将群体构建为"超级个体"）:
  - 核心思想：将每个群体视为一个"超级个体"，构建群体级特征向量，使得可以直接复用个体立场检测方法（如TGAN）进行群体立场预测
  - 特征聚合策略：对群体内筛选后的核心个体，按信息强度加权聚合各维度特征（结构+行为+语义+TGAN嵌入）
  - 群体间交互构建：从原始边表中按群体标签聚合个体间交互，构建群体级"伪边表"，使群体间关系可被图神经网络建模
  - 数据格式对齐：输出格式与原始个体数据完全兼容（节点特征矩阵、边表、边特征、标签），实现"即插即用"

### 3.5 阶段四：群体特征构建与演化分析
- **群体特征聚合**:
  - 均值聚合：组内个体嵌入的平均
  - 注意力聚合：学习个体重要性权重的加权平均
  - 图池化：层次化的图池化方法（如DiffPool、SAGPool）
- **群体交互图构建**:
  - 以群体为超节点，群体间的交互为超边
  - 边权重：群体间的交互频率/强度
  - 边特征：群体间交互的语义特征
- **演化分析(Evolution Analysis)**:
  - **时序立场追踪(Temporal Stance Tracking)**:
    - 将55个时间切片划分为多个时间窗口（如每5个切片为一个窗口，共11个窗口）
    - 在每个时间窗口内，利用TGAN模型的时态嵌入，通过分类器得到每个个体的softmax立场概率（连续立场强度分数）
    - 按群体聚合：计算每个群体在每个时间窗口的平均立场强度
    - 追踪群体立场强度随时间的变化轨迹
  - **交互-演化关联分析(Interaction-Evolution Correlation)**:
    - 在时间窗口t内，统计群体A→群体B的交互量（转发数+回复数）
    - 在时间窗口t+1内，计算群体B的立场变化量（Δstance = stance_{t+1} - stance_t）
    - 分析：交互量与后续立场变化的Pearson/Spearman相关性
    - 关键区分：同立场交互（同党派群体间）vs 跨立场交互（不同党派群体间）的不同影响效果
    - 假设验证：跨立场交互是否导致立场趋同？同立场交互是否强化回音室效应？
  - **群体演化模式分类**:
    - 趋同型(Convergence)：两个群体交互后立场逐渐趋近
    - 极化型(Polarization)：两个群体交互后立场更加对立
    - 稳定型(Stability)：交互不显著影响立场
    - 传染型(Contagion)：单向影响，强势群体改变弱势群体的立场
  - **影响力传播建模**:
    - DeGroot模型：x_i(t+1) = Σ_j w_ij * x_j(t)，群体立场是邻居立场的加权平均
    - Friedkin-Johnsen模型：x_i(t+1) = α_i * s_i + (1-α_i) * Σ_j w_ij * x_j(t)，加入"固执系数"
    - 基于GNN的演化预测：输入t时刻群体特征+交互图，输出t+1时刻群体立场预测（MSE损失）
  - **Granger因果分析**:
    - 对每对群体(A,B)，检验A→B的交互序列是否Granger-causes B的立场变化序列
    - 构建因果影响网络：有向图，边表示显著因果影响
    - 识别核心影响群体：因果网络中出度最高的群体
  - **动态网络演化分析**:
    - 将群体交互图按时间窗口切片，得到时序图序列G_1, G_2, ..., G_T
    - 分析每个时间点的网络拓扑变化：密度、平均路径长度、社区结构
    - 检测关键事件：交互模式的突变点
    - 分析突变前后群体立场的变化
  - **可视化方案**:
    - 演化轨迹图(Evolution Trajectory)：x轴为时间，y轴为立场强度，每条线代表一个群体的立场变化曲线
    - 烟花图(Firework Diagram)：展示同一立场群体的内部信息扩散效果（节点为个体，从核心向外辐射）
    - 演化热力图(Evolution Heatmap)：矩阵形式，行为群体，列为时间窗口，颜色为立场强度
    - 交互-演化联动图(Interaction-Evolution Coupling)：双轴图，上方为交互强度时序，下方为对应的立场变化时序
    - 因果影响网络图(Causal Influence Network)：有向图展示群体间的因果影响关系
    - 小提琴图(Violin Plot)：展示不同时间窗口群体内立场分布的变化
    - 词云图(Word Cloud)：展示不同时间窗口群体话题的演变

### 3.6 阶段五：群体立场分类网络
- **输入**：群体嵌入 + 群体交互图
- **模型架构方案**:
  - GCN/GAT 作用在群体交互图上
  - 群体级别的时态注意力网络
  - 结合个体筛选和群体聚合的端到端模型
- **群体标签定义**:
  - 多数投票：群体内超过半数个体的立场作为群体立场
  - 加权投票：按个体影响力加权的立场聚合
  - 纯度阈值：只对纯度高于阈值的群体赋予标签
- **训练策略**:
  - 群体级别的交叉熵损失
  - 个体级别和群体级别的多任务学习
  - 对比学习：拉近同立场群体、推远异立场群体

## 4. 评估方案

### 4.1 群体划分评估
- 模块度(Modularity)
- 归一化互信息(NMI)
- 群体内立场纯度

### 4.2 群体立场分类评估
- Accuracy, F1-score(macro), AUC-ROC
- 与个体立场分类的性能对比
- 消融实验：验证各组件贡献

### 4.3 演化分析评估
- **群体极化指数(Polarization Index)**：衡量群体间立场对立程度随时间的变化
  - E-I Index时序：每个时间窗口的E-I值，观察极化趋势
  - 交互矩阵模块度：按立场分组后的模块度随时间变化
- **极小化指标(Minimization / Group Merging)**：检验群体合并合理性
  - 群体间交互密度：高密度暗示应合并
  - 特征余弦相似度：高相似度暗示应合并
  - 合并后模块度变化：ΔQ > 0 说明合并有利于整体结构
- **极大化指标(Maximization / Group Splitting)**：检验群体分裂合理性
  - 群体间交互稀疏度：高稀疏度暗示应分裂
  - 特征差异度：高差异度暗示应分裂
  - 分裂后模块度变化：ΔQ > 0 说明分裂有利于整体结构
- **演化预测准确率**：模型预测下一时刻群体立场的准确度（MAE/RMSE）
- **因果显著性**：Granger因果检验的显著性比例（p<0.05的因果对占比）
- **立场稳定性指数**：群体立场时序的方差，方差越小越稳定
- 评价体系总结：极化指数度量对立性演化，极小化/极大化检验群体划分合理性，预测准确率评估模型能力，因果显著性验证影响关系，稳定性指数刻画群体特性
