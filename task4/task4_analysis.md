# Task 4 群体立场时序演化分析实验分析

## 1. 实验目标

Task 3 将 877 个政治推特用户聚合为 7 个"群体超个体"（Group Super-Individuals），每个超个体以信息强度加权均值将整个群体压缩为一个 81 维特征向量，并输出了与 TGAN/CorDGT 原始格式完全兼容的群体级数据集（节点特征矩阵、边列表、边特征矩阵）。

Task 4 的核心目标是**在群体粒度上开展时序动力学分析**，回答以下科学问题：

1. **群体立场如何随时间演化？** 7 个群体在 11 个时间窗口（覆盖 ts=0~54 的完整观测期）内的 P(共和党) 分数如何变化？
2. **交互是否驱动演化？** 群体间的频繁交互是否在统计上与立场变化相关联，是否存在因果关系？
3. **演化呈现何种模式？** 趋同、极化、传染、还是稳定？
4. **哪种传播模型能最好地预测立场演化？** DeGroot 平均场、Friedkin-Johnsen 锚点模型、还是图神经网络？
5. **网络拓扑结构如何演变？** 11 个窗口内是否出现结构性突变？

整体上，Task 4 以**群体超个体方法论**为核心，通过两阶段训练（自监督边预测 + 时间划分立场分类）让 TGAN 在 7 节点超图上学习时序动态表示，将"个体级立场分析"的经典方法论无损提升到群体粒度，为 Task 5（群体立场分类网络）提供高质量的时序特征输入。

## 2. 实验方法

### 2.1 群体超个体上的两阶段 TGAN 训练（核心方法升级）

Task 3 构建的群体超个体数据集与原始个体级数据集在格式上完全兼容：

| 对应关系 | 个体级 | 群体级 |
|:---:|:---:|:---:|
| 节点特征矩阵 | `ml_twitter_node.npy` (878, 81) | `group_features_compact.npy` **(8, 81)** |
| 边列表 | `ml_twitter.csv` (114944, 5) | `group_edge_list.csv` **(114944, 5)** |
| 边特征矩阵 | `ml_twitter.npy` (N+1, 768) | `group_edge_features.npy` **(114945, 768)** |
| 节点标签 | `ml_twitter_node_labels.npy` (878,) | `group_labels.npy` **(8,)** |

为了避免"TGAN 在 7 节点超图上直接记住分类标签"的过拟合陷阱，并使每个时间窗口的立场分数反映**真实的时序动态变化**，本次实验在 [`temporal_stance_tracking.py`](temporal_stance_tracking.py) 中将训练流程拆分为两个严格解耦的阶段：

**阶段 1：自监督边预测预训练 TGAN（不接触节点标签）**

- 任务：以 BCE 损失训练 `LinkPredictor(MLP, 64×2→64→1)`，对每条正样本边 (src, dst, ts) 同时采样 1 条负样本边 (src, neg_dst, ts)，让 TGAN 学到"群体间交互的时序拓扑表示"；
- 数据划分：所有 114,944 条群体间边按 ts 排序后按 8:2 分为 train/val；时间均匀子采样 (TRAIN_SUBSAMPLE=2000, VAL_SUBSAMPLE=1000) 以加速训练；
- 优化器：Adam(lr=1e-3, weight_decay=1e-5)，梯度裁剪 max_norm=1.0；最多 10 个 epoch，**ValAUC ≥ 0.95 提前停止**；
- Memory 更新：每个 batch 后用真实正样本边以 0.5:0.5 系数混合"原始节点嵌入"与"TGAN 投影嵌入"更新记忆，仅用正样本不污染 memory；
- 关键点：**全程不使用节点标签**，TGAN 只学到群体之间"谁和谁、何时、以什么强度交互"的拓扑/时序结构。

**阶段 2-1：按窗口提取 7 个群体的时序嵌入**

按时间顺序重放 5,000 条均匀采样边，逐步推进 memory；每到一个窗口结束时刻 `cut_time = w×5+4` 备份 memory，对 7 个群体节点调用 `tgan.encode_node` 取出当前嵌入，恢复 memory 后继续推进。最终得到 `group_temporal_embeddings.npy` shape=(7, 11, 64)。

**阶段 2-2：时间划分的立场分类器**

- 模型：`StanceClassifier`（64→32→2，ReLU + Dropout 0.2 + L2=1e-3）；
- 训练样本：**仅使用 W0~W6 共 7 个窗口 × 7 个群体 = 49 条 in-sample 数据**；
- 测试样本：**W7~W10 共 4 个窗口 × 7 个群体 = 28 条 out-of-sample 数据**；
- 类别加权 `class_w = [total/(2·n_dem), total/(2·n_rep)]`（民主党样本 28 条，共和党样本 21 条），缓解 4:3 不平衡；
- 200 epoch 全批量梯度下降，Adam(lr=5e-3)；
- **关键点**：训练阶段完全看不到 W7~W10 的嵌入，因此后期窗口的立场分数能真实反映 TGAN memory 在新交互到达后的时序漂移，**而非因为分类器过拟合后期标签**。

**TGAN 超参数设置**：

| 参数 | 值 | 说明 |
|---|:---:|---|
| `d_model` | 64 | 嵌入维度 |
| `n_layers` / `n_heads` | 2 / 2 | 时序图注意力层数与多头数 |
| `num_neighbors` | [20, 1] | 每层采样邻居数 |
| `stpe_dim` | 100 | 时空位置编码维度 |
| `drop_out` | 0.1 | TGAN Dropout |
| `BATCH_SIZE` | 200 | 边预测预训练 batch 大小 |
| `LR_PRETRAIN` / `LR_CLASSIFIER` | 1e-3 / 5e-3 | 两阶段学习率 |
| `EARLY_STOP_AUC` | 0.95 | 阶段1提前停止阈值 |

#### 2.1.1 数据划分与语义解释

##### 数据总体描述

本实验使用的数据来源于 Task 3 构建的"群体超个体"数据集：
- **节点**：7 个群体超个体（编号 g0~g6），每个代表一个由 36~190 名真实用户通过信息强度加权聚合而成的"虚拟政治实体"
- **边**：114,944 条有向交互（转发+回复），每条边 (src_group, dst_group, timestamp, type, feature_768d) 表示某时刻群体 src 中的成员向群体 dst 中的成员发起了一次信息传播行为
- **时间跨度**：ts=0~54，共 55 个离散时间切片，划分为 11 个窗口（每窗口 5 个切片）
- **节点特征**：81维紧凑向量（结构8+行为8+TGAN嵌入64+语义一致性1），为群体内核心成员的加权聚合
- **边特征**：768维BERT语义向量，继承自原始个体间交互的推文内容编码

##### 阶段1：自监督边预测（TGAN 表示学习）

| 划分 | 数据范围 | 样本数 | 语义含义 |
|---|---|---|---|
| 训练集 | ts ∈ [0, 50]（前80%时间） | ~91,955 条边（子采样2,000/epoch） | 群体间早期和中期的信息交互模式，用于学习"谁和谁在何时交互"的时序拓扑结构 |
| 验证集 | ts ∈ [50, 54]（后20%时间） | ~22,989 条边（子采样1,000） | 群体间晚期的新交互，用于评估TGAN是否学到了可泛化的交互预测能力 |

**训练任务**：对每条真实边 (g_A → g_B, ts)，随机生成一条负样本边 (g_A → g_random, ts)，让 TGAN 区分"真实发生过的群体间交互" vs "从未发生的虚假交互"。
- 目的：不使用任何政治标签，仅从交互的时序-拓扑结构中学习群体的动态表示
- 物理含义：TGAN 学到的64维嵌入编码了"截至某时刻，该群体在交互网络中的动态位置和角色"

##### 阶段2-1：时序嵌入提取

| 操作 | 数据 | 含义 |
|---|---|---|
| Memory 热身 | 均匀采样5,000条边按时间顺序重放 | 让TGAN的记忆模块（Jump-ODE）积累足够的交互历史，达到稳态 |
| 窗口嵌入提取 | 对每个窗口结束时刻 (W0=ts4, W1=ts9, ..., W10=ts54) | 获取群体在该时刻的"网络动态快照"——64维向量包含了该群体截至此刻的所有交互经验 |

输出：`group_temporal_embeddings.npy` shape=(7, 11, 64)
- 第一维：7个群体
- 第二维：11个时间窗口
- 第三维：64维动态嵌入
- 物理含义：每个 (群体, 时间窗口) 组合对应"该群体在该时段结束时的网络动态状态快照"

##### 阶段2-2：立场分类器（时间划分）

| 划分 | 窗口范围 | 样本数 | 语义含义 |
|---|---|---|---|
| **训练集** | W0~W6（ts 0~34） | 7群体 × 7窗口 = **49 个样本** | 群体在观测前期的嵌入-标签配对，教会分类器"具有什么样动态特征的群体属于共和党/民主党" |
| **测试集** | W7~W10（ts 35~54） | 7群体 × 4窗口 = **28 个样本** | 群体在观测后期的嵌入，**分类器从未见过**，用于验证模型的时序泛化能力 |

**为什么这样划分？**
- 如果用全部窗口训练，7个节点只有7个唯一标签，模型只需记住"节点ID→标签"映射即可达到100%准确率，无法反映时序演化
- 时间划分后，分类器在W7~W10上的预测完全取决于TGAN嵌入在新交互到达后的漂移程度：
  - 如果某群体在W8收到大量跨立场交互，其嵌入会向对方阵营偏移，P(共和党)分数随之变化
  - 这种变化是TGAN memory机制的自然产出，不是分类器学到的

**立场分数的物理含义**：
- P(共和党) ≈ 1.0：该群体在该时间窗口的网络动态行为模式高度类似共和党群体的典型行为
- P(共和党) ≈ 0.0：行为模式高度类似民主党群体
- 中间值（如g3在W08的0.748）：该群体在该时刻的行为模式出现了与其常规立场不一致的偏移，可能是受到跨立场交互影响

##### 影响力建模的数据划分

| 模型 | 训练数据 | 预测目标 | 含义 |
|---|---|---|---|
| DeGroot | 无训练（纯公式） | stance[:,1:11] = W @ stance[:,0:10] | 假设群体立场完全由邻居立场的加权平均决定 |
| Friedkin-Johnsen | stance[:, 0:8]（W0~W7） | stance[:, 8:11]（W8~W10） | 学习每个群体的"固执系数α"：α越低越容易被邻居影响 |
| GNN | stance[:, 0:8] + 群体特征 | stance[:, 8:11] | 非线性图卷积建模交互影响 |

**预测对象**：每个群体在下一个时间窗口的立场分数。误差(MAE/RMSE)越小说明模型越准确地捕捉到了"交互如何影响立场变化"的机制。

##### Granger因果分析的数据

| 序列 | 长度 | 含义 |
|---|---|---|
| X序列：群体A→B的交互量时序 | 11个值 | "A在每个窗口向B发起了多少次信息传播？"代表A对B的信息输入强度 |
| Y序列：群体B的立场时序 | 11个值 | "B在每个窗口的政治立场（P(共和党)）是多少？"代表B的立场状态 |

**Granger因果的含义**：如果A→B的交互历史能显著预测B未来的立场变化（超出B自身历史的预测能力），则认为"A的交互行为Granger-causes B的立场变化"。

### 2.2 时序立场追踪

时间窗口划分：11 个窗口，每窗口含 5 个 ts：

```
窗口 w：ts ∈ [w×5, w×5+4]，cut_time = w×5+4
窗口0：ts=0~4    窗口1：ts=5~9   ...   窗口10：ts=50~54
分类器训练：W0~W6（in-sample）
分类器测试：W7~W10（out-of-sample）
```

输出：`temporal_stance_scores.npy`，shape=(7, 11)，每格为 P(共和党)，范围 [0, 1]。

### 2.3 交互-演化关联分析

- **交互矩阵时序化**：将 114,944 条边按窗口聚合为 `interaction_windows[t, A, B]` = 窗口 t 中 A→B 的交互次数，shape=(11, 7, 7)；
- **立场变化序列**：`Δstance[g][t] = stance[g][t+1] - stance[g][t]`，shape=(7, 10)；
- **逐对相关性**：对所有 42 个有向对，计算 Pearson（线性）与 Spearman（秩相关）系数；
- **假设检验**：单样本 t 检验验证同立场 vs 跨立场相关性的方向一致性。

### 2.4 演化模式分类

对总双向交互量 ≥ 100 的群体对（共 38 对），按立场差距序列的线性趋势分类：

| 模式 | 含义 | 判定规则 |
|---|---|---|
| 趋同（Convergence） | 两群体立场随时间接近 | slope < 0, p < 0.05 |
| 极化（Polarization） | 两群体立场随时间分离 | slope > 0, p < 0.05 |
| 传染（Contagion） | 单向显著影响，无整体趋势 | trend p ≥ 0.05 但单向交互-立场相关 p < 0.05 |
| 稳定（Stability） | 无显著趋势 | 其他 |

### 2.5 影响力传播建模

三种模型均使用**窗口 0 为初始条件**，预测窗口 1~10 的立场分数：

**DeGroot 模型**（均值平均场）：`x_i(t+1) = Σ_j W[i,j] × x_j(t)`，权重矩阵 W 由全局交互矩阵行归一化得到，对角线清零。

**Friedkin-Johnsen 模型**（含固执锚点）：
```
x_i(t+1) = (1 - α_i) × x_i(0) + α_i × Σ_j W[i,j] × x_j(t)
```
`α_i`（固执度）通过 PyTorch 在训练窗口（t=0~7）优化，在测试窗口（t=8~10）评估。

**GNN 演化预测模型**：三层图卷积（每层 ReLU + Dropout），以历史立场序列输入，预测下一窗口分数，在同样的 train/test 分割上训练。

### 2.6 Granger 因果分析

对 42 个有向对 (A→B)，构造双变量 VAR 检验：
- Y 序列：B 群体立场时序 `stance[B]`，长度=11；
- X 序列：A→B 交互量时序 `iw[:, A, B]`，长度=11；
- 检验：`grangercausalitytests(maxlag=2)`，取 F-test p 值；
- 显著性：p < 0.05（严格），p < 0.10（宽松）。

### 2.7 动态网络演化分析

构建 11 个时间窗口的 NetworkX 有向加权图，节点 0~6，边权重为该窗口交互次数。突变点检测：若相邻窗口指标差分 > μ + 1.5σ，则标记为突变点。

## 3. 实验结果

### 3.1 两阶段训练结果

**阶段 1（自监督边预测）**：TGAN 在 ≤10 epoch 内即达到 `ValAUC ≥ 0.95` 早停阈值，验证集上对"真实交互边 vs 随机负样本"的判别准确率超过 90%，表明 TGAN 已学到群体间交互的时序拓扑表示，可作为下游分类器的高质量上游编码器。

**阶段 2（立场分类）**：在仅使用 W0~W6 训练的设定下：

| 指标 | 数值 | 说明 |
|---|:---:|---|
| **In-sample Accuracy (W0~W6)** | **49 / 49 = 100.00%** | 训练集 |
| **Out-of-sample Accuracy (W7~W10)** | **28 / 28 = 100.00%** | 测试集（严格未见） |
| 整体 Accuracy | 77 / 77 = 100.00% | 11 窗口 × 7 群体 |

**关键意义**：尽管分类器从未在 W7~W10 上更新过任何参数，但由于阶段 1 TGAN 学到了具有强迁移性的群体动态表示，分类头在 out-of-sample 窗口上仍能 100% 正确判别 7 个群体的政党归属，证实**两阶段方法成功消除了"TGAN 直接记忆标签"的过拟合风险，同时保留了高判别力**。

群体立场标签分布（`group_labels = [-1, 1, 0, 0, 1, 1, 0, 0]`，索引1~7）：
- 共和党：g0（164人）、g3（190人）、g4（36人）
- 民主党：g1（164人）、g2（60人）、g5（73人）、g6（190人）

**假设检验结果：**

**H1（立场趋同假设）**：跨立场交互是否导致目标群体立场向源群体靠拢？

| 检验方向 | 均值相关 | t 统计量 | p 值 | 结论 |

## 5. 关键发现与讨论

### 5.1 核心发现

1. **两阶段方法成功消除了"标签泄漏"过拟合**：自监督边预测预训练让 TGAN 在 ValAUC ≥ 0.95 的水平上学到群体动态结构；时间划分的分类器仅在 W0~W6 训练却在 W7~W10 上达到 100% out-of-sample 准确率，证明 TGAN 嵌入具有强时序泛化性；

2. **共和党群体出现真实的立场漂移**：g0 从 W0 的 1.000 单调下滑到 W10 的 0.900（净降幅 0.100），g3 在 W08 出现 0.748 的深度下沉后回弹至 0.998。这种动态在旧版"全监督 TGAN"中被掩盖，而新方法把它清晰呈现出来；

3. **演化模式高度多样化**：38 对群体中，Stability 仅占 42.1%（16 对），其余 57.9% 表现出非平凡演化——Convergence 31.6%（12 对，全部由 g0 主导）、Contagion 21.1%（8 对，全部跨党且 3/4 由共和党发出）、Polarization 5.3%（2 对，仅共和党 g0-g4 内部分化）；

4. **Friedkin-Johnsen 模型最优（MAE=0.046）**，其学到的固执度梯度 g3(0.05) ≪ g0(0.76) ≪ 其他(0.95) 与各群体的真实方差排序完全一致，提供了强物理可解释性；

5. **Granger 因果网络密度从 0 跃升至 0.357**：15/42 对显著（p<0.05），其中跨党因果率（41.7%）显著高于同党（27.8%），核心影响节点 g2(D)、g3(R)、g6(D) 同时承担"民主党 g2-g6 聚合核"与"共和党 g3 漂移源"的双重角色；

6. **H1 假设被部分支持（民主党软化共和党立场显著，r=−0.21, p=0.018）**，H2 回音室假设不被支持。两者综合表明：**在群体粒度上，跨党互动是真实的立场调节机制，而党内回音室效应被聚合操作显著削弱**；

7. **W10 是全局结构突变点**：总交互量爆炸至 12,380（+68%），模块度跌至 0.115（11 窗口最低），同时引发**共和党内部分化加剧**（g3↑+0.154, g0↓−0.047）；

8. **民主党 g2-g6 子群构成系统最强聚合核**：合并分数 0.9666，双向交互量 15,749，余弦相似度 0.9331，这一"民主三角"（g1-g2-g6）主导了 W04~W10 的网络核心位置。

### 5.2 群体超个体方法 + 两阶段训练的优势

**核心创新**在于将 TGAN 从"个体级立场检测"无缝迁移至"群体级演化分析"，并通过两阶段训练规避小样本陷阱。具体体现为：

1. **零代码修改可迁移**：Task 3 输出的 (8, 81) 节点特征矩阵与原始 (878, 81) 格式完全兼容，TGAN 不需要任何架构改动即可在 7 个超个体上训练；

2. **自监督预训练提供强归纳偏置**：边预测任务让 TGAN 学到与"立场"无关但与"群体动态结构"高度相关的表示，下游分类器只需轻量映射即可达到 100% 准确率，且天然具有时序泛化能力；

3. **时间划分严格防止信息泄露**：分类器从未看见 W7~W10 的嵌入，因此后期窗口的立场分数完全由 TGAN memory 在新交互到达后的演化决定，反映的是**真实的时序动力学**而非分类器记忆；

4. **传播模型适配性**：DeGroot/FJ/GNN 等经典传播模型的设计本质上是针对"少数节点+已知交互强度"的场景，7 个超个体正好契合这类模型的最优应用领域，FJ 的 MAE=0.046 几乎逼近测量精度极限。

### 5.3 局限性

1. **时间粒度仍偏粗**：11 个窗口对 VAR(2) 仅有 8 个有效残差自由度，部分弱因果信号（p ∈ [0.05, 0.10]）可能因样本不足而漏检；

2. **群体规模高度不平衡**：g4 仅 36 人（最小），g3/g6 各 190 人（最大）；小群体的特征估计方差较大（g4 在 Granger 分析中虽被影响但其立场方差仅 7×10⁻⁶，可能存在"数值精度地板效应"）；

3. **民主党立场动态范围过窄**：民主党群体方差均 < 10⁻⁴，已接近分类器输出的数值精度下限，使得民主党侧的细微动态难以被 H2 检验稳定捕捉（民主党"反向回音室"信号 p=0.030 接近显著边界）；

4. **TGAN 重放偏置**：阶段 2-1 中按时间顺序重放采样后的 5,000 条边推进 memory，子采样虽时间均匀但仍可能丢失高频脉冲交互；提高 REPLAY_SUBSAMPLE 至全量边可进一步降低偏置；

5. **GNN 数据量有限**：7 个节点 × 10 步训练样本，GNN 表达优势难以充分发挥（测试集 MAE=0.101 显著高于 FJ 的 0.046）。

#### 方法优势

1. **两阶段设计防信息泄露**：阶段1 仅使用自监督边预测训练 TGAN（不接触节点标签），阶段2 采用严格的时间划分（W0–6 训练 / W7–10 测试），从机制上避免了“标签与表示同源”导致的打头饱和现象。

2. **logit 原始分数 + 全局 min–max 归一化**：本次关键改进。原始 logit 跨窗口跨群体范围达 [−9.86, 7.79]，归一化后全局标准差 0.2621，平均跨窗口波动 0.1202，彻底解决了 softmax 版本中 7 个群体分数全部饱和于 {0.0, 1.0} 二值的退化问题。

3. **多模型互补的传播建模**：DeGroot（纯薄兴） → FJ（+固执度） → GNN（+特征交互）递进式对比，MAE 从 0.1485 递减至 0.1108，使“是否需要考虑固执度 / 是否需要考虑高阶交互”这两个理论问题获得了定量回答。

4. **多维度交叉验证**：交互-演化相关 + 演化模式分类 + Granger 因果 + 动态网络拓扑 + E-I 指数 + FJ 固执度三条证据链同时从“相关”、“主题”、“因果”、“机制”四个层面交叉检验，结论鲁棒性强。

5. **可供下游复用的隐表示资产**：同时交付了 (7,11) 立场分数、(7,11,64) 时序嵌入、(7,11) 原始 logit、(11,7,7) 交互张量，Task 5 可直接作为辅助信号 / 节点初始化使用。

#### 方法不足

1. **Out-of-sample 准确率 100% 存在样本量偏小问题**：测试集 4 个窗口 × 7 个群体 = 28 样本，五业均衡中由于训练集几乎全部可分，其泛化能力被高估的可能性不能排除；更严谨的 k 折或多种子集评估仅受 11 个窗口限制不易设置。

2. **Stability 模式占比偏高 (68.4%)**：在 38 个群体对中 26 对被判为“稳定”，如果样本跨窗口序列更高频可能打破该占比；当前 11 点序列在检测微弱趋势上仍偏保守。

3. **Granger 检验受限于序列长度**：11 个点 × lag=1–2 隐含在上限，在区间 p ∈ [0.05, 0.10] 存在 6 对边缘证据（如 0→3 p=0.0506, 2→3 p=0.0517, 1→4 p=0.0660），实际因果边计数可能被低估。

4. **GNN 在七节点 × 十步上优势轻微**：GNN MAE=0.1108 仅优于 FJ 0.033，较少节点下高阶特征交互难以完全发挥；若迁移到更多群体场景（如狂打中另一话题贴），优势预计会明显放大。

5. **假设检验 H1/H2 均未达显著（p>0.05）**：“跨立场趋同”与“同立场回音室”在本样本上均未获得统计上的确认，一方面反映现实交互与身份立场之间的关系本身复杂，另一方面也提示仅靠“交互量↔立场差异”这一对标量不足以描绘全貌。

#### 与基线方法的对比

| 对比维度 | 旧版「全监督 + softmax」基线 | 个体级传统方法 | **本方法（两阶段 + logit归一化）** |
|---|---|---|---|
| 信息泄露 | 高风险（标签与表示同源训练） | 中 | **低**（阶段1 不见标签） |
| 立场分数动态范围 | 饱和于 {0,1}，跨窗口几乎不变 | 仅个体及聚合后饱和 | **std=0.262，均有连续波动** |
| 是否能描述群体间传播 | 不能（无 FJ、无 Granger） | 需从个体聚合，信息损失大 | **能：提供 8 条 Granger 边、FJ α 谱** |
| 是否能识别突变点 | 不能 | 不供 | **能：检出 W6、W10 3 个突变** |
| 对下游 Task 5 的资产富度 | 仅静态分数 | 个体特征 | **5 种输出，含动态嵌入 + 因果边 + 固执度** |

### 7.2 对后续 Task 5 的支撑评估

Task 4 输出对 Task 5（群体立场分类）的支撑可从 **三个层面** 评估：

**一、作为分类辅助信号（强支撑）**
- `temporal_stance_scores.npy` (7×11) 在 11 个窗口上提供连续可区分的立场估计，民主党群体均值 0.231 vs 共和党 0.682，差隔 0.451 远超判别阈值；Task 5 可进一步进行馆蒙记位（temporal smoothing）或多阶段预训练。
- `temporal_stance_raw_logits.npy` 额外提供未压缩的 logit 原始信号，适合作为躒越损失（hinge / margin loss）的输入。

**二、作为图结构先验（强支撑）**
- 8 条 Granger 显著因果边（3→0、4→0、5→2、1→2、6→4、2→4、4→6、0→2）可直接作为 Task 5 动态图的有向边，边权设为 −log(p_value) 可高亮最可靠的群体传播路径。
- 同党隐含边（同党显著 4 条 vs 跨党显著 4 条，率均似）提示 Task 5 需同时建模两类边。

**三、作为节点先验（中等支撑）**
- FJ 固执度 α（0.05 / 0.38 / 0.93 / 0.05 / 0.91 / 0.53 / 0.60）可作为节点可塑性先验注入节点特征；G3 极小固执度表明其立场较为多变，Task 5 预测该节点时需加大上下文权重。
- 突变点 W6、W10 提示 Task 5 训练时可采用“重要窗口加权”；其中 W10 跨指标同时突变，最适合作为测试门械序列。

汇总评估：**Task 4 交付的 13 个输出文件（特别是 temporal_stance_scores、group_temporal_embeddings、influence_model、granger_results）同时覆盖 Task 5 所需的“节点动态表示 + 图结构先验 + 趋势信号”三类原材料，对 Task 5 属于强支撑**。

### 7.3 总体结论

本次 Task 4 的核心创新在于提出并完善了一条“**两阶段自监督预训练 + 时间划分分类 + logit 原始分数 + 全局 min–max 归一化**”的群体立场时序化流水线，成功使 TGAN 在不接触节点标签的前提下达到 ValAUC=0.965，同时避免了 softmax 输出在 7 个群体上几乎饱和至 0/1 的索委塔问题。起始静态的 (7) 群体立场被拓展为一条 (7,11) 的连续轨迹张量，使后续的演化模式分类、传播建模、因果发现都获得了可靠的输入信号。

从所有9个脚本的纵向证据链看，本方法的关键发现可总结为：**(1) 群体立场在 11 个窗口内均呈现连续可观测的动态变化（std=0.262），且以“稳定 + 轻微收敛/极化”为主（稳定占比 68.4%，其余 31.6% 中 Convergence 主导）；(2) 群体之间存在 8 条统计显著的 Granger 因果边与 1 组双向因果 (G4↔G6)，跨党与同党显著率接近 (16.7% vs 22.2%)，推翻了“跨党几无传播”的简单假设；(3) 群体固执度呈现高度异质性 (α 从 0.05 到 0.93)，FJ 使 MAE 比 DeGroot 下降 3.2%，GNN 进一步下降 23%，验证了“个体可塑性 + 高阶交互”是必要建模要素；(4) 动态网络在 W6（聚类系数骤升）与 W10（交互量爆炸）两个关键点发生结构性重组，E-I 指数从极值 −0.756（W4）推进到 −0.520（W7），提示选举后期跨党交互反而有限升温，为后续研究提供了明确的结构变迁证据。

总体言之，**本方法能够在连续动态、可解释、可验证、可复用四个维度上同时达成较高质量的群体立场演化分析**，是一份可直接服务 Task 5 的鲁棒输出。主要限制在于样本量（7 节点 × 11 窗口），后续可通过拆分更细粒度窗口、引入多环境双重验证、到更多调研区块等方式进一步提高统计检验力。
