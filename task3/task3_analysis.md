# Task 3 强信息个体筛选与群体特征整合实验分析

## 1. 实验目标

在 Task 2 给出的 7 个平衡群体（共 877 个有效个体）基础上，完成两件事：

1. **强信息个体筛选**：通过"活跃度 + 影响力 + 信息量"三维度评分，筛选每个群体的高信息量核心个体（共 524 人，覆盖 59.75%）；
2. **群体特征整合（"超级个体"）**：以信息强度评分为权重，将每个群体聚合为一个"超级个体"，形成 (8, 81) 紧凑特征矩阵与 (8, 849) 完整特征矩阵；同时把 114,944 条个体级交互映射为群体级交互，并构造与原始 `ml_twitter_node.npy` / `ml_twitter.csv` / `ml_twitter.npy` **格式完全兼容**的群体级数据集，便于直接灌入 TGAN / CorDGT 等下游立场检测模型做"群体立场预测"。

整体上，Task 3 起到把"个体级"立场分析数据无缝转换为"群体级"立场分析数据的桥梁作用。

## 2. 实验方法

### 2.1 个体信息强度评分（`individual_scoring.py`）

#### 2.1.1 三维度评分体系

| 维度 | 权重 | 计算公式 | 数据来源 |
|---|:----:|---|---|
| 活跃度（Activity） | **0.40** | `tweet_count × active_time_span` | `behavioral_features_raw.npy` 第 0、5 列 |
| 影响力（Influence） | **0.35** | `in_degree + pagerank × 1000` | `structural_features.npy` 第 0、4 列 |
| 信息量（Information） | **0.25** | `1 ? semantic_consistency` | `semantic_features.npy` 最后一列 |

权重设计直觉：

- **活跃度（0.40）** 最高，因高活跃用户为群体特征提供最多观测；
- **影响力（0.35）** 次之，捕捉网络中心度与传播能力（PageRank × 1000 是为了把数量级与 in-degree 对齐）；
- **信息量（0.25）** 较低但不可缺，反映用户语义内容的多样性，避免"高活跃但内容单一"的机器人/转推站被过度加权。

#### 2.1.2 标准化方法

每个维度先单独做 **Min-Max 标准化** 到 `[0, 1]`：

```
x_norm = (x ? min) / (max ? min)
```

最终：`composite_score = 0.4 × activity + 0.35 × influence + 0.25 × info`

### 2.2 强信息个体筛选策略

```
对每个群体 g ∈ {0, 1, ..., 6}:
    按 composite_score 降序排列群体成员
    n_keep = max(int(n_members × 0.6), min(15, n_members))
    保留 Top-n_keep 个个体（filtered_mask=True）
```

- **保留比例**：每群体 Top-**60%**；
- **最小保留**：每群体不少于 **15** 人，防止小群体（如 g=4 仅 36 人）过度收缩；
- **筛选依据**：综合评分 `composite_score`。

### 2.3 群体特征聚合（`group_feature_integration.py`）

#### 2.3.1 加权策略

仅在筛选后的核心个体（524 人）上，用其 `composite_score` 作为权重做加权平均：

```
weights[m] = score[m] / Σ score[member]   # 归一化
group_feat[g] = Σ_m weights[m] × individual_feat[m]
```

若某群体权重和过小，则退化为均匀权重 `1 / n_members`。该策略相比"简单均值"的优势：

- 高分个体支配群体表征，避免"沉默用户"稀释信号；
- 同时保留低分个体的微弱贡献，防止特征向单点崩塌。

#### 2.3.2 聚合维度

| 块 | 维度 | 来源 | 备注 |
|---|:----:|---|---|
| 结构特征（Structural） | 8 | `structural_features.npy` | in/out/total degree、closeness/betweenness/clustering、pagerank、reaching |
| 行为特征（Behavioral） | 8 | `behavioral_features.npy` | tweet_count、retweet/reply 比、mention 频率、活跃时长、内容多样性、长度等 |
| TGAN 嵌入 | 64 | `tgan_embeddings.npy` | Task 1 用 TGAN 学到的动态网络嵌入 |
| 语义一致性 | 1 | `semantic_features.npy[-1]` | 群体内语义凝聚度 |
| 完整语义（BERT） | 768 | `semantic_features.npy[:768]` | 仅在 full 矩阵中拼接 |

最终输出：

- `group_features_compact.npy`，shape = **(8, 81)** = 8 + 8 + 64 + 1
- `group_features_full.npy`，shape = **(8, 849)** = 8 + 8 + 768 + 1 + 64

两矩阵第 0 行均为占位零向量（与 `ml_twitter_node.npy` 中索引 0 留空的约定一致），群体 g（0..6）对应矩阵第 g+1 行。

### 2.4 群体间交互构建

#### 2.4.1 边映射

对原始 `ml_twitter.csv` 的 114,944 条个体级边，做向量化映射：

```
g_u = group_labels[u];  g_i = group_labels[i]
保留 (g_u, g_i) 均有效的边
```

筛选后保留 **114,944 条群体级有向边**（所有原始边均落入 7 个群体，无丢失）。每条群体级边继承原始边的 `ts` 与 `label`，`idx` 重新从 1 编号。

#### 2.4.2 边特征映射

群体级边的 768 维 BERT 特征直接从原始 `ml_twitter.npy` 按 `idx` 取出，使用 **memmap 分块写入**（chunk=5000）以适配 2GB 内存约束：

```python
out_mmap = open_memmap(edge_feat_path, mode='w+', dtype=float32, shape=(N+1, 768))
# 第 0 行保持零，群体级边 i 取原始边 idx[i] 的 768 维特征
```

输出 `group_edge_features.npy`，shape = **(114,945, 768)**。

#### 2.4.3 群体交互矩阵

构建 7×7 整数矩阵 `interaction_matrix[src, dst] = #{edges with g_u=src, g_i=dst}`，shape=(7, 7)，总和 = 114,944。

### 2.5 群体立场标签生成

采用**多数投票法**：

```
对群体 g:
    n_dem = #{members with party=0}
    n_rep = #{members with party=1}
    label = 0 (Dem) if n_dem >= n_rep else 1 (Rep)
```

注意：投票基于**全部群体成员（877 人原始群体）**，而非筛选后的 524 人，以保证立场标签反映完整的群体构成。

输出 `group_labels.npy`，shape=(8,)，索引 0 为占位 `-1`，索引 1..7 为群体 0..6 的立场。

## 6. 结论与讨论

### 6.1 群体作为"超级个体"的合理性

1. **维度无损**：紧凑版 81 维与个体特征完全同构，下游模型零修改可用；
2. **信息抽取保真**：通过 60% Top-K 筛选 + 信息强度加权，过滤了沉默用户与噪声节点，保留群体核心信号；
3. **结构一致**：群体级交互完全继承原始时间戳与交互类型，时序模型（TGAN）的"动态信息"在群体粒度依然成立；
4. **立场可解释**：多数投票生成的群体标签与 Task 2 评估的纯度高度一致（大部分群体 ≥ 95%）。

### 6.2 信息强度加权 vs 简单均值

- **简单均值**：所有成员等权重，沉默用户与活跃核心同等贡献，会弱化群体特征区分度；
- **信息强度加权**：高活跃 / 高影响 / 高语义多样性的核心成员主导群体表征，使得 7 个群体在结构维度上的差异（如 g2 vs g5 的 `in_degree` 81.75 vs 4.59）被显著放大，这种差异正是下游分类任务最关键的信号；
- 同时，加权并非"硬选 Top-K 后求均"，而是"软加权"，所有筛选后的成员都对均值产生贡献，避免了硬阈值带来的不连续性。

实验上，群体 5 这种内部高度异质的桥接群体，加权后语义一致性仅 0.522（远低于其他群体 0.86–0.90），如实保留了其多元性 —— 这是简单均值难以呈现的细节。

### 6.3 局限性

1. **群体规模差异显著**：g3、g6 各有 190 人，g4 仅 36 人。即便经过加权，小群体的特征估计方差仍较大；
2. **跨立场桥接信号被压缩**：群体 0（22 民 + 142 共）经多数投票后被打成纯共和党，22 名民主党成员的"边界"语义被吸收到群体特征里，但 label 上消失；
3. **三维度评分权重为人工设定**：0.4/0.35/0.25 是基于先验经验，未做网格搜索 / 学习；
4. **TGAN 嵌入冷启动问题**：若群体内大部分成员的 TGAN 嵌入来自早期训练阶段，群体特征可能偏向陈旧时序；
5. **群体级交互未做时间窗聚合**：`group_edge_list.csv` 包含 11.5 万条边，对极小群体（如 g5）来说交互极稀疏（仅 556 条自交互），下游模型可能需要额外的样本增强。

### 6.4 后续改进方向

- **学习式权重**：用立场分类性能反向优化"活跃 / 影响 / 信息"三维权重；
- **多粒度立场标签**：除多数投票外，附加"民主党比例"软标签，保留少数派信号；
- **超级个体的时序版本**：在不同时间窗口分别聚合，得到群体的演化轨迹（衔接已有的"群体立场演化分析"工作）；
- **跨群体边的方向重要性建模**：将"跨立场交互"作为单独通道喂给 GNN，强化对极化 / 桥接现象的建模能力。
